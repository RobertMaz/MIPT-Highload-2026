## Задание

### Часть 1: Deployment-документ

Берёте шаблон с семинара от 30.04, применяете к проектному кейсу.

#### 1.1. Описание развёртывания

- Краткое описание системы и нагрузки
- Deployment diagram (draw.io → PNG): сервисы с replicas, AZ, network zones, направление связей
- Список сервисов: ответственность, stateful/stateless, public/private

#### 1.2. Стратегия деплоя

Для ключевых сервисов:

- Стратегия (Recreate / Rolling / Canary) и почему именно она под профиль сервиса
- Zero-downtime контракт: probes (readiness/liveness), graceful shutdown
- Подход к миграциям БД (Expand/Contract или почему не применимо)

#### 1.3. Observability

- 4 алерта на critical path по Golden Signals: метрика, порог, окно, на какой сервис
- Дашборды по 3 уровням (Overview / Service / Diagnostic) — что на каждом
- Логи: формат, обязательные поля, что логируем, что не логируем

### Часть 2: Доступность

- Целевая доступность системы (99.9% / 99.95% / другое) — обоснование от бизнеса: сколько денег теряется при простое, что ожидают
  пользователи
- RPO и RTO для критичных компонентов (БД, очередь, кэш — у каждого свои требования к восстановлению)
- Стратегия резервирования для них: active/active или active/standby, нужна ли геораспределённость
- Maintenance window: есть ли вообще, какое окно, как часто

### Часть 3: TCO в Yandex Cloud

Калькулятор: https://yandex.cloud/ru/prices

Считайте только то, что реально есть в вашей архитектуре. Мапинг типичных компонентов на сервисы YC:

| Компонент              | Сервис YC                                 |
|------------------------|-------------------------------------------|
| Compute (микросервисы) | Compute Cloud (VM) или Managed Kubernetes |
| PostgreSQL             | Managed Service for PostgreSQL            |
| Redis / KV-кэш         | Managed Service for Redis                 |
| Kafka / очереди        | Managed Service for Apache Kafka          |
| Объектное хранилище    | Object Storage                            |
| Балансировщик          | Application Load Balancer                 |
| Мониторинг             | Yandex Monitoring + Cloud Logging         |

Посчитайте:

- Стоимость инфраструктуры на текущую нагрузку (compute + storage + network + managed) — ₽/мес
- Как меняется при росте трафика в 2x и 5x
- Операционные затраты (мониторинг, on-call, обслуживание) — оценка/мес

Результат: таблица с расчётами + комментарий «что дороже всего и почему». Точность до рубля не нужна — важен порядок величин.
